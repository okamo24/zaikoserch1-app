# 在庫検索アプリ MVP要件書

## 1. 目的
本システムは、委託先倉庫で利用する在庫検索業務を効率化するための Web アプリである。  
既存のクラプロ UI を参考にするが、全機能を再現するのではなく、必要機能のみに絞って構築する。  
MVP では、在庫、メンバー、CSV インポート、設定の 4 機能を対象とする。

## 2. 対象ユーザー
- `admin`
  - システム管理者
  - メンバー承認、権限変更、CSV インポートを行う
- `user`
  - 一般利用者
  - 在庫検索を利用する
  - メンバー一覧の参照のみ可能

## 3. 前提
- UI はクラプロを参考にするが、不要機能は持ち込まない
- スマホ、PC ともにチャット型 UI で統一する
- MVP 段階では実装範囲を最小限に絞る
- 実データに存在しない項目は表示対象としない
- 認証は Google ログインを利用する

## 4. 対象機能
- 在庫
- メンバー
- CSV インポート
- 設定

## 5. 在庫機能要件
- スマホ、PC ともにチャット型 UI とする
- 画面上部に以下のサマリーを表示する
  - 総在庫数
  - 総品目数
  - 最終取込日時
- 入力手段は以下を提供する
  - 文字入力
  - 音声入力
  - カメラ読取
- 検索対象は以下とする
  - 商品名
  - 商品コード
  - JAN
  - ITF
- 検索結果はチャット形式で表示する
- 1 件ヒット時は結果カードを表示する
- 複数件ヒット時は候補一覧を表示する
- 結果カードをタップすると詳細画面を表示する

## 6. 在庫一覧表示項目
- 商品名
- 商品コード
- 入数
- ロケ
- 在庫数
- JAN またはバーコード相当の識別情報

## 7. 在庫詳細表示項目
- 商品名
- 商品コード
- 入数
- ロケ
- バーコード
- 在庫数
- 在庫基準日
- 最終取込日時

## 8. 在庫表示除外項目
- 仕入価格
- 販売価格

## 9. メンバー機能要件
- `admin` は以下を利用可能とする
  - メンバー一覧表示
  - 承認状態確認
  - 承認日確認
  - 承認、非承認管理
  - 権限変更
  - 論理削除
- `user` は以下のみ利用可能とする
  - メンバー一覧表示

## 10. メンバー管理項目
- 名前
- メールアドレス
- 権限
- 承認状態
- 承認日
- 作成日
- 最終ログイン日時
- 論理削除状態

## 11. CSV インポート機能要件
- `admin` のみ利用可能とする
- 取込対象は以下の 2 ファイルとする
  - 在庫 CSV
  - ロケ CSV
- 2 ファイルが揃っている場合のみ取込実行可能とする
- 取込実行後に検索用データを更新する
- 画面上で以下を確認できること
  - 最終取込日時
  - 取込結果
  - 取込件数
  - エラー件数
  - エラー内容
  - 取込履歴一覧

## 12. CSV インポート履歴表示項目
- 取込日時
- 取込者
- 在庫 CSV 件数
- ロケ CSV 件数
- 成功件数
- エラー件数
- ステータス

## 13. 設定機能要件
- ログイン中アカウントを表示する
- テーマ色は項目のみ表示し、機能は未実装とする
- 問合せフォームは項目のみ表示し、機能は未実装とする

## 14. 認証、権限制御
- Google ログインを利用する
- ログイン後、未承認ユーザーは利用不可とする
- 利用可否は `admin` が承認管理する
- `admin` と `user` で利用可能機能を制御する

## 15. MVP 対象外機能
- レポート
- アラート
- 作業
- 取引先
- 場所
- バーコード印刷
- プラン表示
- クラプロ固有の課金、契約関連機能
- テーマ色の実変更
- 問合せフォーム送信機能

## 16. UI 方針
- クラプロ UI は参考とするが、必要機能のみ採用する
- 在庫画面はチャット型 UI を維持する
- PC 版でもスマホ版と同じ操作思想を採用する
- 不要な情報や機能は持ち込まない
